# 📋 모두의 연수등록부 (All-in-One Training Register)
> 창순기획 | Vite + GitHub Pages + Google Apps Script (Sheets as DB)

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 모두의 연수등록부 |
| 목적 | 학교 연수·협의회 등의 출석 및 정보 수집을 디지털화 |
| 프론트엔드 | Vite (Vanilla JS) — Antigraviton 스택, GitHub Pages 배포 |
| 백엔드/DB | Google Apps Script + Google Sheets |
| 주요 사용자 | 관리자(교사·담당자), 일반 사용자(연수 참여자) |

---

## 2. 핵심 기능 요약

### 👤 사용자 흐름
1. 관리자가 공유한 링크로 접속
2. 연수명 선택 (드롭다운)
3. 이름 입력
4. 관리자가 설계한 **모듈 박스** 내용 입력
5. 제출 → Google Sheets에 저장

### 🛠 관리자 흐름
1. 관리자 페이지 로그인 (비밀번호 or Google 계정)
2. 연수 생성 → 모듈 박스 드래그&드롭 배치
3. 공유 링크 생성
4. 응답 데이터 확인 → CSV / PDF 다운로드

## 5. 서명 모듈 상세 사양 (핵심)

### 5-1. 목표 UX
- 실제 볼펜으로 종이에 서명하는 **부드럽고 자연스러운 필기감**
- 모바일(터치) / 데스크탑(마우스) 모두 동일 품질 지원

### 5-2. 기술 구현

```javascript
// Canvas 기반 서명 구현 핵심 요소

// 1. 필압 시뮬레이션 (속도 → 선 굵기 반비례)
const getLineWidth = (speed) => {
  const minWidth = 0.8;
  const maxWidth = 3.5;
  return Math.max(minWidth, maxWidth - speed * 0.3);
};

// 2. 베지어 곡선으로 부드러운 선 처리
const drawSmoothLine = (ctx, points) => {
  ctx.beginPath();
  ctx.moveTo(points[0].x, points[0].y);
  for (let i = 1; i < points.length - 1; i++) {
    const midX = (points[i].x + points[i + 1].x) / 2;
    const midY = (points[i].y + points[i + 1].y) / 2;
    ctx.quadraticCurveTo(points[i].x, points[i].y, midX, midY);
  }
  ctx.stroke();
};

// 3. 이벤트: 터치 + 마우스 통합 처리
canvas.addEventListener('pointerdown', startDraw);
canvas.addEventListener('pointermove', draw);
canvas.addEventListener('pointerup', endDraw);
```

**pen 스타일 설정:**
```javascript
ctx.strokeStyle = '#1a1a2e';   // 진한 남색 (볼펜 느낌)
ctx.lineJoin = 'round';
ctx.lineCap = 'round';
ctx.globalAlpha = 0.92;        // 약간의 투명도로 자연스러운 잉크 번짐 표현
```

### 5-3. 서명 입력 방식 (이중 지원)

| 방식 | 설명 |
|------|------|
| **터치패드/마우스 직접 서명** | Canvas에 직접 서명 (기본) |
| **이미지 파일 업로드** | 스캔한 서명 이미지 업로드 (PNG/JPG) → Canvas에 렌더링 후 동일하게 저장 |

**이미지 업로드 처리:**
```javascript
// 업로드된 서명 이미지 → Canvas 렌더링 → Base64 저장 (통일된 포맷)
const handleSignatureUpload = (file) => {
  const reader = new FileReader();
  reader.onload = (e) => {
    const img = new Image();
    img.onload = () => {
      ctx.clearRect(0, 0, canvas.width, canvas.height);
      // 비율 유지하며 Canvas에 맞게 그리기
      const ratio = Math.min(canvas.width / img.width, canvas.height / img.height);
      const x = (canvas.width - img.width * ratio) / 2;
      const y = (canvas.height - img.height * ratio) / 2;
      ctx.drawImage(img, x, y, img.width * ratio, img.height * ratio);
    };
    img.src = e.target.result;
  };
  reader.readAsDataURL(file);
};
```

### 5-4. 서명 데이터 저장
- Canvas → `canvas.toDataURL('image/png')` → **Base64 문자열**로 Sheets에 저장
- PDF 출력 시 해당 Base64를 이미지로 복원하여 삽입

## 7. Google Apps Script (백엔드) 설계

### 7-1. Sheets 구조

| 시트명 | 역할 | 주요 컬럼 |
|--------|------|-----------|
| `trainings` | 연수 목록 | id, 연수명, 날짜, 장소, 모듈설정JSON, 생성일 |
| `responses_{id}` | 연수별 응답 | 타임스탬프, 이름, 모듈별 응답값... |
| `admins` | 관리자 계정 | id, 비밀번호(해시), 권한 |

### 7-2. API 엔드포인트 (doGet / doPost)

```javascript
// Code.gs 라우팅 구조
function doGet(e) {
  const action = e.parameter.action;
  switch(action) {
    case 'getTrainings':   return getTrainingList();
    case 'getForm':        return getFormConfig(e.parameter.id);
    case 'getResponses':   return getResponses(e.parameter.id);
    default:               return error('Unknown action');
  }
}

function doPost(e) {
  const data = JSON.parse(e.postData.contents);
  switch(data.action) {
    case 'submitResponse':  return submitResponse(data);
    case 'createTraining':  return createTraining(data);
    case 'updateForm':      return updateFormConfig(data);
    case 'deleteTraining':  return deleteTraining(data);
    case 'adminLogin':      return adminLogin(data);
    default:                return error('Unknown action');
  }
}
```

### 7-3. 응답 저장 방식
- 서명 데이터는 Base64 전체 저장 (Sheets 셀은 최대 50,000자 → 초과 시 Google Drive 이미지로 저장 후 URL 참조)
- 모든 응답은 JSON 직렬화 후 전송 → GAS에서 파싱하여 행 단위로 저장

## 11. GitHub Pages 배포 설정

```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run build
      - uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```
